• 🔥14 **Anthropic 更新 Claude 政策：允許提供「公開可得」武器資訊** — Anthropic——以 AI 安全為核心賣點、估值 615 億美元的公司——悄然更新了 Claude 的使用政策，允許模型在資訊「已公開可得」的前提下，回應涉及武器、爆炸物及危險材料的問題。這是一次重大的路線轉向：Dario Amodei 和 Daniela Amodei 當初正是以「不夠重視安全」為由離開 OpenAI，創立 Anthropic。批評者認為「公開可得」是個過於寬鬆的標準，幾乎任何資訊都能以此為由通過。
📎 批評視角：AI 安全組織 MLSF 和多位研究員指出，「資訊已公開」不等於「AI 協助組織化使用」不會造成額外傷害；若 Claude 能整合、簡化危險步驟，危害性遠高於讓使用者自行搜尋。這也可能給監管機構提供更多介入理由。

## 🛡️ 資安威脅

• 🔥13 **Trivy 安全掃描器供應鏈攻擊：75 個 GitHub Actions 標籤被劫持，CI/CD 密鑰外洩** — 廣泛使用的開源漏洞掃描工具 Trivy 遭到供應鏈攻擊，攻擊者組織 hackerbot-claw 利用 `pull_request_target` 工作流程漏洞竊取 Personal Access Token（PAT），進而劫持 GitHub repository，篡改 75 個 release 標籤以注入惡意程式碼。受損版本（v0.69.4）存活約三小時，但被篡改的 Actions 標籤維持活躍長達 12 小時，期間可能竊取了使用這些 Actions 的 CI/CD pipeline 密鑰。
📎 批評視角：這起事件再次暴露開源供應鏈的系統性脆弱：數千個生產環境信任 Trivy 的 Actions 標籤，但幾乎沒有機制驗證這些標籤的完整性。企業應立即固定 Actions 版本至 commit hash，而非使用可被篡改的浮動標籤。
📎 https://thehackernews.com/2026/03/trivy-security-scanner-github-actions.html
📎 https://www.bleepingcomputer.com/news/security/trivy-vulnerability-scanner-breach-pushed-infostealer-via-github-actions/

• 🔥11 **Oracle 緊急修補 Identity Manager 嚴重 RCE 漏洞（CVE-2026-21992，CVSS 9.8）** — Oracle 發布緊急更新修復 Identity Manager/Web Services 中的未驗證遠端程式碼執行漏洞，CVSS 評分 9.8（滿分 10），屬最高危險等級。任何暴露於網路的 Oracle Identity Manager 實例都應立即套用補丁。
📎 https://www.hendryadrian.com/cybersecurity-news-daily-recap-21-mar-2026/
