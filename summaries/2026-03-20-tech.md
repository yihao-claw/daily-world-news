## 🧠 LLM

• 🔥15 **OpenAI 收購 Python 開發工具公司 Astral** — OpenAI 今日宣布將收購 Astral，整合其廣受開發者喜愛的開源 Python 工具：套件管理工具 uv、超高速 linter/formatter Ruff，以及型別檢查工具 ty，融入 Codex AI 開發平台生態系。Astral 的工具每月被數百萬開發者使用，此次收購讓 OpenAI 直接掌控 Python 開發流程的基礎設施層。
  ⚠️ **批評視角**：Hacker News 社群擔憂：「原本的開放軟體堆疊被 AI 巨頭收購後，還能繼續維持開源嗎？」業界觀察者指出，此舉延續了 AI 公司壟斷開發工具的趨勢，使開發者更深度依賴單一平台。
  📎 https://www.investing.com/news/company-news/openai-to-acquire-astral-boosting-codex-developer-tools-93CH-4570840
  📎 https://simonwillison.net/2026/Mar/19/openai-acquiring-astral/

• 🔥13 **Xiaomi MiMo-V2-Pro：1 兆參數 LLM，1M token 脈絡，挑戰 Claude 與 GPT** — 小米於 3 月 19 日深夜發布三款 MiMo-V2 新模型：旗艦推理模型 MiMo-V2-Pro（1T 參數、42B 活躍參數、MoE 架構、1M token 脈絡窗口）、多模態 MiMo-V2-Omni（支援文字/圖片/視頻/音訊），以及 MiMo-V2-TTS 情感語音模型。VentureBeat 報導 MiMo-V2-Pro 在 PinchBench 評測以 84.0 分名列全球第三（僅次 Claude 4.6 兩款），且 API 費率顯著低於同等級競品。
  ⚠️ **批評視角**：Xiaomi 使用自家 PinchBench 基準測試，外部獨立驗證尚未完成；分析師提醒「1T 參數」中實際每次推理只啟動 42B，與大眾直覺的「1 兆參數模型」存在落差，需等待第三方評測。
  📎 https://venturebeat.com/technology/xiaomi-stuns-with-new-mimo-v2-pro-llm-nearing-gpt-5-2-opus-4-6-performance
  📎 https://www.gizmochina.com/2026/03/19/xiaomi-unveils-mimo-v2-pro-its-flagship-llm-with-over-1tb-of-parameters/

• 🔥10 **Apple「LLM in a Flash」技術：讓 Qwen3.5-397B 在本機 SSD 串流執行** — Simon Willison 利用 Andrej Karpathy 的 AutoResearch 系統，驗證了 Apple 研究論文提出的「LLM in a Flash」技術——透過將 MoE 模型的 expert 參數從 SSD 串流載入記憶體，實現在消費級硬體上執行 397B 參數模型。這開啟了本地大型 MoE 推理的新可能，不需要全量 VRAM。
  📎 https://simonwillison.net/2026/Mar/18/llm-in-a-flash/

## 📈 趨勢分析

本週科技圈三大主軸：

1. **AI 公司搶佔開發工具控制權** — OpenAI 收購 Astral 延續近期 AI 大廠收購開源工具趨勢（類似微軟收購 GitHub），目標是讓開發者在 AI 輔助編碼時更深度留在自家平台。開放性 vs. 商業壟斷的張力將持續升溫。

2. **中國 AI 追趕加速** — Xiaomi MiMo-V2-Pro 顯示中國科技公司在 LLM 能力上快速縮小差距，且以更低 API 費率參與全球競爭。繼 DeepSeek 之後，小米、百度、阿里的模型正在構成真正的競爭壓力。

3. **AI Agent RL 訓練成為下一個戰場** — Deeptune 融資、Karpathy AutoResearch、Microsoft RL for tool-use agents 等訊號同步出現，業界正從「靜態訓練」轉向「動態 RL 訓練場」。2026 將是 Agentic AI 的真正落地年，而其訓練基礎設施的投資競爭才剛開始。
