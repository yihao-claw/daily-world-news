## 🧠 LLM

• 🔥13 **Hugging Face Transformers v5.6.0 釋出：內建 OpenAI Privacy Filter**
  主版號 5.6.0 引進 OpenAI 官方贊助的雙向 token-classification PII 遮罩模型，專為高吞吐量的資料淨化流程設計，讓企業能在不上傳雲端的前提下自動遮蔽姓名、地址、信用卡等個資。這是 Transformers 首次把主要雲端廠牌的隱私過濾器直接收進核心庫。
  📎 來源：[huggingface/transformers v5.6.0](https://github.com/huggingface/transformers/releases/tag/v5.6.0)

• 🔥13 **Ollama v0.21.1：Kimi CLI 直接整合、MLX runner 新增 logprobs**
  可用 `ollama launch kimi --model kimi-k2.6:cloud` 一鍵裝 Moonshot 的 Kimi CLI；Apple Silicon 的 MLX runner 這版加入 logprobs 與更快的取樣。昨天還是 rc1，今天正式推出。
  🔻 風險視角：Kimi K2.6 的長時程 agentic 執行能力強，但中國廠牌雲端 API 的資料出境議題在 EU/美國企業部署仍卡關。
  📎 來源：[ollama/ollama v0.21.1](https://github.com/ollama/ollama/releases/tag/v0.21.1)

• 🔥10 **Anthropic「Claude Mythos Preview」早期版協助 Firefox 150 修掉 271 個安全漏洞**
  Mozilla 工程師 Bobby Holley 公開：這週釋出的 Firefox 150 裡，有 271 個漏洞是靠 Anthropic 早期 Mythos Preview 模型協助評估找出的，是史上單一 Firefox 版本修補最多 CVE 的紀錄之一。Holley 強調：團隊「單一目標、全力衝刺」才看得到隧道盡頭的光。
  🔻 批評視角：271 個漏洞乍聽驚人，但 CVE 嚴重度分布未公開，多數可能是低風險記憶體誤用；AI-assisted triage 仍需要人工複核，否則會塞爆 upstream patch pipeline。
  📎 來源：[Simon Willison](https://simonwillison.net/2026/Apr/22/bobby-holley/#atom-everything)

• 🔥10 **Gary Marcus 連發兩文質疑 ChatGPT Images 2.0：「連攪拌器和手肘都分不清」**
  這是 OpenAI 上週發表 Images 2.0（2K 輸出、Thinking 模式、連續 8 張）後的最新批評：Marcus 舉出多個醫學解剖與廚房工具圖例，ChatGPT 生成結果與題目完全對不上；他的結論是「Regurgitating ≠ understanding」，醫療插畫師短期內不會被取代。
  📎 來源：[Gary Marcus – whisk vs elbow](https://garymarcus.substack.com/p/chatgpt-doesnt-know-its-whisk-from) | [Gary Marcus – Image engine](https://garymarcus.substack.com/p/chatgpts-powerful-new-image-engine)

• 🔥10 **「Over-editing」：研究者量化 LLM 改動超出必要範圍的行為**
  `nrehiew.github.io/blog/minimal_editing/` 定義 over-editing — 模型在應該只改 3 行時改了 30 行，在 HN 衝到 261 分、142 留言。作者建議以 `diff_lines / required_lines` 作為 agentic coding 的新評分維度，和目前主流的 pass@k 互補。
  🔻 反方：AI 工程圈普遍認為 over-editing 是可控的 prompt engineering 問題，把它變成 benchmark 指標會鼓勵模型「保守到底」，反而失去 agentic workflow 的優勢。
  📎 來源：[Hacker News](https://news.ycombinator.com/item?id=47866913) | [原文](https://nrehiew.github.io/blog/minimal_editing/)
