## 🧠 LLM 與模型發展

📎 批評視角：RC 版本仍屬測試階段，大型企業通常需等待穩定版才能用於生產環境，實際效益需數月後驗證。
📎 https://github.com/pytorch/pytorch/releases/tag/v2.11.0-rc6

• 🔥11 **[GitHub Release] Ollama v0.18.2 發布** — 本地端 LLM 運行工具推出新版，修復了 model 選取問題（#14943），提升本地端 AI 部署的穩定性，讓開發者更容易在自己的硬體上運行大型語言模型。
📎 https://github.com/ollama/ollama/releases/tag/v0.18.2

• 🔥 **LinkedIn 以單一 LLM 取代五套推薦系統** — LinkedIn 宣布完成一項架構大重構：用一個 LLM 模型，取代了原本針對 13 億用戶的五套獨立 feed 推薦管線。工程師花費一整年設計，最終使複雜度大幅降低。
📎 批評視角：將如此關鍵的推薦系統集中於單一模型，增加了「單點故障」風險，且 LLM 的決策可解釋性遠低於傳統 ML 系統，廣告主難以審計推薦邏輯。
📎 https://venturebeat.com/orchestration/how-linkedin-replaced-five-feed-retrieval-systems-with-one-llm-model-at-1-3

• 🔥 **HBR 研究：向 LLM 問策略建議，得到的是「趨勢爛料」（Trendslop）** — 哈佛商業評論新研究發現，企業向 ChatGPT 等 LLM 尋求戰略建議時，往往獲得充滿管理學流行語但缺乏具體洞見的回應，研究者稱之為「Trendslop（潮流廢話）」。
📎 批評視角：這直接挑戰了「AI 諮詢師」的商業模式——若 LLM 的策略建議品質不如資深顧問，企業花費在 AI 整合上的預算需要重新評估。
📎 https://hbr.org/2026/03/researchers-asked-llms-for-strategic-advice-they-got-trendslop-in-return

📎 https://www.helpnetsecurity.com/2026/03/20/connectwise-screenconnect-cve-2026-3564/

• **DarkSword iOS 漏洞利用工具包曝光：鎖定 iPhone 零點擊攻擊** — 安全研究人員發現新的 iOS 漏洞利用框架「DarkSword」，已鎖定多個 WebKit 和 dyld 漏洞實施零點擊攻擊，部分漏洞此前已被 Apple 緊急修補。
