# 品質規範

## 科技新聞評分公式

### 基礎分 = 5

### 加分項
| 條件 | 分數 | 說明 |
|------|------|------|
| 優先來源 (priority: true) | +3 | TECH_SOURCES.json 中標記為 priority |
| 多來源交叉報導 | +5 | 同一事件被 2+ 不同來源報導 |
| 時效性（24h 內）| +2 | 過去 24 小時內發布 |
| 高互動（Reddit > 500）| +3 | Reddit 高分帖子 |
| 中互動（Reddit > 200）| +1 | |
| GitHub 高星（daily > 50）| +3 | GitHub Trending 熱門 |

### 減分項
| 條件 | 分數 | 說明 |
|------|------|------|
| 昨日已報導 | -5 | 與前一天摘要重複 |
| 低品質來源 | -2 | 來源不在 TECH_SOURCES.json 中 |

**最低發布分 = 5**

## 時事新聞優先級（非數字評分）

1. 重大突發事件（影響多國）
2. 多國媒體報導（越多國越重要）
3. 對讀者直接影響（尤其日本/台灣/東亞）
4. 時效性（今日 > 持續 > 背景）
5. 獨特性（與昨日不重複）

## 去重規則

### Dedup Gate（必做）
寫任何摘要前，讀過去 **3 天**的摘要，對每個候選故事做 **New Development Test**：
- **有新事實/數字/反應** -> 正常納入
- **同題微更新（如價格變動 1%）** -> 壓縮為一句或合併
- **無新進展** -> 直接刪除
- **連續 3 天無實質變化** -> 必須刪除

### 科技去重
- 標題相似度 > 85% -> 合併，保留最權威來源
- 同一 GitHub repo 多個小更新 -> 合併為一則
- RSS + Reddit + Web Search 報同一事 -> 合併，加多來源分

### 時事去重
- 同一事件只有「重大新進展」才納入
- 多國報導同一事件 -> 不算重複，整合為格式 A

### 跨區塊去重
- 時事版保留政治/政策角度，科技版保留技術角度
- 只有一個角度 -> 放最合適的區塊

## 反面觀點要求（Counter-Perspective Mandate）

### 時事新聞
- 每則重大新聞（格式 A）**必須包含至少一個反對觀點或批評聲音**
- 來自不同國家、立場或利益相關方
- 格式：`🔻 [來源/立場]：[觀點摘要]`
- 禁止只呈現單一敘事

### 科技新聞
- 每個重大 LLM/AI Agent 進展需加一句「批評或風險視角」
- 加密貨幣報導需同時提及看多和看空觀點，附具體數字佐證

---

## 數據佐證要求（Data Evidence Mandate）

涉及經濟、軍事、科技、氣候的報導**必須附具體數字**：
- 「大幅增長」-> 「同比增長 12.3%（IMF 數據）」
- 「傷亡慘重」-> 「已造成 47 人死亡（聯合國人道協調廳）」
- 數據來源需標注機構縮寫（IMF, UN, WHO, WTO, WMO, OECD 等）
- 若暫無官方數據，寫「數據待核實」，不得杜撰數字

---

## 市場數據日期校正（必做）

`market.json` 中每個指標有 `date` field。必須遵守：

1. **檢查每個指標的 `date`**，判斷數據實際來自哪一天
2. **對照 `$TODAY` 是星期幾：** 週末/假日股市休市
3. **休市就說休市，不重複舊數字：** 前一天已經報過了
4. **只播報有新數據的市場：** 週末通常只有加密貨幣和外匯
5. **不同市場數據日期可能不同：** 美股可能是週五、日股可能是週四，不能混在一起用「都漲了」概括
6. **加密貨幣 24/7 交易，** 日期通常較即時，可以用「目前」

## 事實核查流程

### 時事核查（Step 2.55）
1. **人名 & 職稱** — web_search 確認
2. **具體數字** — 必須與 📎 來源一致，用 web_fetch 抽查
3. **事件日期 & 時序** — 相對於 $TODAY 是否正確
4. **因果關係** — 不要隱含來源沒有宣稱的因果
5. **引述歸屬** — 引用必須歸對人/媒體

### 科技核查（Step 1.45）
- **版本號** — 確認是 v4.0 還是 v3.5
- **模型 benchmark** — 不要杜撰分數，web_fetch 來源確認
- **融資金額/估值** — 確認金額和輪次
- **發布狀態** — 區分 released / announced / coming soon
