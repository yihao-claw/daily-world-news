# 新聞摘要產出格式規範

## 基本要求

- **語言**：繁體中文
- **每日產出**：1 份完整摘要，存為 `summaries/YYYY-MM-DD.md`
- **推送平台**：Telegram（不使用 markdown 表格）

## 地區分類（9 大區塊）

總計 **15-20 則精選新聞**，依當日新聞重要性分配各區塊數量（不需要每區都有）。

1. 🇺🇸 **美洲**（美國、加拿大、拉美）
2. 🇪🇺 **歐洲**（歐盟、英國、北歐、東歐）
3. 🇯🇵🇰🇷🇨🇳 **東亞**（日本、韓國、中國、香港、蒙古）
4. 🇹🇼 **台灣**
5. 🇮🇳 **南亞 & 東南亞**（印度、巴基斯坦、東協各國）
6. 🇸🇦 **中東**（以色列、伊朗、海灣國家、土耳其）
7. 🌍 **非洲**
8. 🇦🇺 **大洋洲**（澳洲、紐西蘭、太平洋島國）
9. 🌐 **科技 & 財經**（全球科技、金融市場、加密貨幣、AI）

## 每則新聞格式

### 格式 A：多元視角（重大國際事件）

適用於：涉及多國利益的重大事件（如中美關係、中東衝突、台海、貿易戰等）

```
• **[標題/事件名稱]**
  [事件摘要 1-2 句]

  🇺🇸 美國：[該國媒體的報導重點、措辭、立場]
  🇨🇳 中國：[該國媒體的報導重點、措辭、立場]
  🇯🇵 日本：[該國媒體的報導重點、措辭、立場]
  ...（依事件相關國家列出，不預設分組）

  💡 觀點差異：[一句話點出各國報導的關鍵分歧]
  📎 來源：（每個網址獨立一行，不用 markdown 超連結）
  https://example.com/article1
  https://example.com/article2
```

### 格式 B：標準格式（地區性 / 單一國家新聞）

```
• **[標題/事件名稱]** — [2-3 句深入說明，包含背景脈絡、影響]
  📎 https://example.com/article
```

### 使用原則

- **格式 A**：15-20 則中，建議 3-5 則使用多元視角（最重大的國際事件）
- **格式 B**：其餘新聞使用標準格式
- 按**國家**列出觀點，不要用「東方/西方」等粗糙分類
- 只列**跟該事件有直接利害關係的國家**

## 深度要求

每則新聞的說明必須包含：

1. **事件本身** — 發生了什麼
2. **背景脈絡** — 為什麼重要、前因後果
3. **影響分析** — 對誰有影響、市場/政治/社會的反應
4. **多元視角**（格式 A）— 不同國家媒體的報導角度、措辭差異、省略了什麼

## 整體結構

```markdown
# 🌍 每日世界新聞摘要 — YYYY年M月D日（週X）

> 📡 資料來源：[列出主要參考媒體]
> ⏰ 更新時間：UTC HH:MM

---

## 🔥 今日頭條（Top 3）
[當日最重大的 3 則新聞，跨地區，附來源]

---

## 🇺🇸 美洲 / 🇪🇺 歐洲 / ... (各區塊)

---

## 📊 市場快照
- 美股：S&P 500 / Nasdaq / Dow（漲跌幅）
- 歐股 / 亞股 / 匯率 / 原油 / 黃金 / 加密貨幣
```

## 來源要求

- 每則新聞必須附上至少一個原始來源連結
- 來源優先使用 [config/SOURCES.md](../config/SOURCES.md) 中列出的媒體
- 重大事件可附多個來源交叉比對
- 不得使用無法驗證的來源或編造新聞內容

## 新聞數量目標

- **總計**：15-20 則精選（質 > 量）
- **今日頭條**：Top 3（從各區塊中挑出最重大的）
- **各區塊**：依當日新聞重要性動態分配，不強制每區都有
- **去重複**：同一事件被多家媒體報導時，合併為一則，整合多方觀點與來源

## Telegram 推送注意事項

- 不使用 markdown 表格（Telegram 不支援）
- 使用 bullet points + bold
- 若總長度超過 4096 字元，分多則發送（在 ## 邊界切割）
